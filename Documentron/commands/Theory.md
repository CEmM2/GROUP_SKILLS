# Theory

Build, check, refresh, or map scientific and mathematical documentation.

## Modes

- `build`: create missing theory surfaces from verified evidence.
- `check`: read-only correctness review.
- `refresh`: update invalidated claims only.
- `map`: update deterministic theory-to-code/test/context relationships.

## Workflow

1. Resolve theory documents and their mapped source/tests; run `prepare --command theory`.
2. Deterministically check links, equation delimiters, symbol/glossary coverage, cited paths, units fields, tolerances, and configured executable validations.
3. Always route invalidated scientific semantics through the theorist plus matched domain lenses.
4. Run a fresh scientific reviewer when the packet requires independent review.
5. For build/refresh, have the writer emit preimage-hashed patches; for check/map, remain read-only.
6. Validate results, apply permitted patches, re-run preparation, and render reports.

Read `references/scientific_review.md`. Distinguish `implemented`, `partially-implemented`, `planned`, `context-only`, `unsupported`, and `unknown`. Do not auto-rewrite equations from general knowledge.
