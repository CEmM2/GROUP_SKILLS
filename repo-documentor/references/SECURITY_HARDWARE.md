# Security, safety, and hardware notes

Include this section for:
- motor control / embedded
- lab acquisition / experimental tooling
- any system interacting with real devices

Document:
- Hardware assumptions (models, interfaces, OS constraints)
- Safety limits (current, speed, temperature, travel, force)
- Interlocks, watchdogs, emergency stop behavior
- Failure modes and safe shutdown path
- Calibration procedures and required frequency
- Logging and traceability artifacts (and where they are stored)
